📘 Objetivos del Proyecto Integrador por Semestre

📌 Objetivo general: Diseñar y construir una base sólida en lógica, estructuras de datos, diseño web y modelado de bases de datos.
  • Base de Datos: Diseñar el modelo entidad-relación y construir la base de datos en MySQL.
  • Introducción al Desarrollo Web: Maquetar interfaces con HTML semántico, estilos CSS y frameworks como Bootstrap o Tailwind.
  • Lógica de Programación: Diseñar e implementar algoritmos utilizando estructuras de control, arreglos y objetos en Java.
📌 Objetivo general: Desarrollar aplicaciones dinámicas que conecten el frontend con la base de datos usando JavaScript y Programación Orientada a Objetos.
  • Frontend 1: Manipular dinámicamente el DOM con JavaScript puro, aplicar eventos y efectos visuales.
  • Backend 1: Modelar problemas y conectar con la base de datos usando POO en Java.
  • Metodologías Ágiles: Comprender e identificar los elementos de Scrum: roles, artefactos y eventos.
📌 Objetivo general: Integrar los componentes de software en una arquitectura moderna, desarrollando una API backend, un cliente SPA y habilidades en análisis de datos.
  • Backend 2: Crear una API REST bajo el patrón de capas utilizando Spring Boot.
  • Frontend 2: Construir una aplicación SPA en React, consumiendo servicios REST y renderizando componentes de forma dinámica.
  • Nuevas Tecnologías: Analizar fuentes de datos con Python y generar reportes mediante herramientas de visualización.

💡 Metodologías Activas para Ideación y Cocreación

Las etapas de ideación y cocreación en proyectos formativos son momentos clave donde se construyen conceptos, se exploran soluciones y se fomenta la creatividad colectiva. En este contexto, las metodologías activas permiten que los estudiantes asuman un rol protagónico, desarrollando pensamiento crítico, empatía y habilidades de comunicación.

Basados en enfoques como Design Thinking, estas metodologías impulsan la creación colaborativa desde la exploración de necesidades hasta la propuesta de soluciones funcionales. A continuación, se destacan algunas estrategias aplicadas:

  • 🧱 Lego® Serious Play®: Herramienta poderosa para representar conceptos abstractos a través de modelos físicos, fomenta el pensamiento visual, la narrativa y el trabajo en equipo.
  • 🧩 Gamificación y Escape Room: Juegos estructurados con retos progresivos que estimulan el razonamiento lógico, la resolución de problemas y la colaboración bajo presión.
  • 🎮 Juegos Online: Plataformas lúdicas o simuladores que permiten tomar decisiones, aplicar conocimientos y reflexionar sobre errores en entornos digitales seguros.
  • 🎨 Design Thinking: Metodología centrada en el usuario que estructura la ideación en cinco fases: empatizar, definir, idear, prototipar y testear. Ideal para fomentar el pensamiento creativo con propósito.
Estas estrategias no solo dinamizan el aula, sino que mejoran la retención del conocimiento, fortalecen las competencias del siglo XXI y generan ambientes de aprendizaje emocionalmente significativos.
📸 Evidencias de Aplicación
🧱 Lego® Serious Play®

Técnica visual y colaborativa para representar ideas, modelar algoritmos y simular estructuras de pensamiento lógico. Fortalece habilidades blandas y permite identificar cómo los estudiantes conceptualizan la lógica de forma tangible.

✅ Aplicación exitosa:

En nivel 1, los estudiantes construyeron algoritmos con piezas Lego, identificando tipos de datos y estructuras de control. Se promovió el diálogo técnico en lenguaje cotidiano.

📌 Recomendaciones por nivel:
  • Nivel 1 (Lógica): Representar condicionales, ciclos y estructuras de datos simples con modelos físicos.
  • Nivel 2 (POO): Modelar objetos, relaciones y jerarquías entre clases.
  • Nivel 3 (API): Representar arquitectura por capas y flujos de petición-respuesta con piezas.


🧩 Escape Room y Estrategias Lúdicas

Actividades de gamificación que convierten el aula en un escenario de retos secuenciales. Los estudiantes resuelven pistas, problemas y acertijos que integran contenidos técnicos, promoviendo la motivación, la lógica, la cooperación y la toma de decisiones bajo presión.

✅ Aplicación exitosa:

En nivel 1, se diseñó un Escape Room digital con 50 preguntas integradas sobre lógica, ciclos, tipos de datos y operadores en Java. Los estudiantes participaron por equipos, resolviendo cada fase con base en su análisis colectivo, desarrollando pensamiento lógico y comunicación efectiva.

📌 Recomendaciones por nivel:
  • Nivel 1 (Lógica): Crear retos de programación que usen condicionales, bucles y operadores como claves para avanzar.
  • Nivel 2 (Frontend / POO): Incorporar pistas basadas en código HTML/JS y estructuras de objetos en problemas prácticos.
  • Nivel 3 (API / Datos): Diseñar misiones donde deben interpretar respuestas JSON de una API, o analizar datos simulados para tomar decisiones.
